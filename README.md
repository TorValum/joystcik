# joystcik
🤖 Контекст проекта для ИИ-ассистентаПроект: Программный комплекс ручного управления БПЛА (самолетного типа).Текущий этап: Реализация и отладка протокола связи между наземной станцией управления и бортовым контроллером.Структура системы:Наземная станция (Python): Скрипт использует pygame для захвата данных с джойстика и pyserial для отправки команд в UART.Бортовой компьютер (STM32 / C / HAL): Прошивка принимает данные, парсит их и управляет сервоприводами через ШИМ (TIM3, TIM4).

Используемый протокол:Передача идет строками, заканчивающимися на \n.Формат команды: <символ_команды> <числовой_аргумент>.На стороне STM32 реализовано накопление в буфер rx_buffer и разбор через sscanf.

Логика управления:Команды 1-4 (Тангаж/Крен): Используют arg для плавной установки значений ШИМ (1000–2000).Команды 9, 8 (Парашют): Срабатывают по кнопкам, аргумент игнорируется, значения ШИМ фиксированные (2000/1500).Команда 0: Нейтральное положение (1500).

Что требуется от ИИ:Соблюдать текущую структуру функции Process_Radio_Commands (сброс флага ORE, буферизация, sscanf).При доработке Python-кода учитывать формат отправки f"{cmd} {arg}\n".Помогать с реализацией алгоритмов стабилизации и безопасности (Fail-safe).
